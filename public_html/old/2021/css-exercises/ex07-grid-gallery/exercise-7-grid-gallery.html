Exercise 7

This is Exercise #7, lecture on CSS Grid, used in the course IDATA2301 Web technologies at NTNU, campus Aalesund.

Go back to exercise list.

Purpose

The purpose of the exercise is to practice the implicit behaviour of CSS grids: automatically-generated columns and rows.

Instructions

Use this document as a template. You see a set of product cards below. Make the container a grid and configure the following behaviour for it:

  1. The grid should have automatic columns, 240px wide.
  2. The grid should have automatic rows, 400px high.
  3. The grid should try to fill as many cards as possible per row.
  4. The container should not shrink to less than 500px horizontally (hint: min-width property).

Hints

You probably want to configure several things (just using grid-auto-columns will not be enough, the grid will decide that it is enough with a single column!):

  • Tell that the grid should repeat creating as many columns of necessary size as can fit within the available width.
  • Specify the height of auto-created rows.

Extra challenge - alignment

To do this, your will need to know the other part of the lecture, see "Item Alignment" in the slides.

The task : center the cards within the grid cells. If you look at the current grid, the default alignment is not nice - the cards are stretched vertically, there is not space between them. Also, if you look at the grid (hover over the container in Chrome developer tools), you will notice that the cards are aligned on the left of each cell. Put them in the center both horizontally and vertically!
